# BM25

BM25 (Best Match 25) is a probabilistic ranking function that improves on TF-IDF by solving two fundamental problems: unbounded term frequency and no awareness of document length.

## Problems with TF-IDF

**1. TF grows linearly — but relevance doesn't**

In plain TF-IDF, if a term appears 100 times in a document it scores 100× higher than a document where it appears once. But a document isn't 100× more relevant just because it repeats a word more. After a few occurrences, additional repetitions carry diminishing informational value.

**2. Long documents are unfairly rewarded**

A 10,000-word document will naturally contain more occurrences of any term than a 100-word document — not because it's more relevant, but simply because it's longer. TF-IDF has no mechanism to account for this.

---

## How BM25 Fixes This

### TF Saturation

BM25 replaces raw TF with a saturating version:

```
bm25_tf = tf * (k1 + 1) / (tf + k1 * length_norm)
```

As `tf` grows toward infinity, `bm25_tf` approaches a ceiling of `k1 + 1`. This means the 100th occurrence of a word contributes almost nothing compared to the 1st — which matches intuition.

The **k1** parameter controls how quickly saturation kicks in:
- Lower k1 → faster saturation (diminishing returns start sooner)
- Higher k1 → slower saturation (each additional occurrence still matters more)
- Typical value: `1.5`

### Length Normalization

BM25 adjusts for document length by computing:

```
length_norm = 1 - b + b * (doc_length / avg_doc_length)
```

This factor is plugged into the TF denominator. A document longer than average gets a higher `length_norm`, which reduces its TF score — penalizing it for its length.

The **b** parameter controls how aggressively length is penalized:
- `b = 0` → no length normalization (pure TF saturation only)
- `b = 1` → full normalization (document length fully factored in)
- Typical value: `0.75`

### Why Log in IDF

BM25 IDF is:

```
bm25_idf = log((N - df + 0.5) / (df + 0.5) + 1)
```

where `N` is the total number of documents and `df` is how many documents contain the term.

The log matters because without it, a term appearing in 1 out of 1,000,000 documents would score 1,000,000× higher than a term in 500,000 documents — an astronomically disproportionate difference. The log compresses that scale so rare terms are meaningfully but not absurdly more important than common ones.

The `+ 0.5` smoothing in numerator and denominator prevents division by zero and handles edge cases (a term in every document, or in no document). The outer `+ 1` ensures IDF is always positive — without it, a term appearing in more than half the documents would yield a negative IDF and drag scores down.

### Full BM25 Score

The final score for a query term in a document is simply:

```
bm25(doc, term) = bm25_tf(doc, term) * bm25_idf(term)
```

For multi-term queries, scores are summed across all query tokens.

---

## Parameters Reference

| Parameter | Default | Effect |
|-----------|---------|--------|
| `k1` | `1.5` | TF saturation speed — higher = slower saturation |
| `b` | `0.75` | Length normalization strength — `0` to `1` |
